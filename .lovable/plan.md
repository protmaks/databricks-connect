План исправления всплывающих панелей

1. Исправить логику ширины панелей
- Сейчас `AiMatchesPanel` после поиска открывается как фиксированная панель 340px, из-за чего на экране 571px она выглядит как 1/4–1/3 доступной области, а не как 1/2.
- Сделаю так, чтобы панель результатов поиска занимала нормальную читаемую ширину: на узких экранах почти всю рабочую область, на широких — ограниченную удобную ширину.

2. Исправить режим “две панели открыты”
- Сейчас `AgentDetailPanel` получает `offsetRight="50%"` и `w-1/2`, но это ломается относительно `main`, особенно при наличии левого фильтра шириной 288px: панель физически смещается так, что видна только часть.
- Вместо двух независимых absolute-панелей с процентными offset сделаю общий контейнер справа для панелей.
- Когда открыта одна панель — она занимает весь контейнер.
- Когда открыты две панели — контейнер делится на две колонки 50/50, и обе панели полностью видны.

3. Сделать адаптивное поведение для маленькой ширины
- Для текущего viewport 571px с левым sidebar места слишком мало для двух колонок рядом.
- На узком экране панели будут открываться как overlay поверх карты/части sidebar или занимать рабочую область так, чтобы текст не обрезался.
- Если две панели открыты на узком экране, они не будут уходить за экран: либо будут stacked/scrollable, либо будут занимать всю доступную ширину с корректной прокруткой.

4. Обновить компоненты
- `Index.tsx`: заменить текущую схему `offsetRight={agentPlan ? "50%" : 0}` и `compact` на более надежный layout wrapper.
- `AiMatchesPanel.tsx`: убрать жесткую absolute-привязку `right-0` из самого компонента или сделать ее управляемой снаружи.
- `AgentDetailPanel.tsx`: убрать проблемный percentage-offset и сделать ширину зависимой от родительского контейнера.

5. Проверить ожидаемый UX
- После AI поиска: панель результатов полностью видна и читаема.
- После клика по facility: две панели открыты рядом без обрезания.
- На маленьком экране: ни одна панель не видна “наполовину”; контент доступен через нормальную прокрутку.

Техническая идея
```text
main
 ├─ map
 └─ panels overlay container
     ├─ search results panel  (1 column if alone, 1/2 if enough space)
     └─ facility detail panel (1/2 if enough space)
```

После одобрения внесу изменения в React/Tailwind layout без изменения backend/database.
